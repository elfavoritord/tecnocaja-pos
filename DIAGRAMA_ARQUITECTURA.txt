🔗 FLUJO COMPLETO (De principio a fin)
═══════════════════════════════════════════════════════════════════════════════

1. USUARIO CREA VENTA EN TECNO_CAJA
   └─ Cliente compra producto → Se registra en MariaDB local ✓
   └─ POS sigue funcionando 100% normal (sin internet)

2. VENTA SE COMPLETA Y "PAGA"
   └─ Evento: "venta_completada"
   └─ Firebase Sync Service lo detecta
   └─ Se agrega a cola local (sqlite)

3. SINCRONIZACIÓN AUTOMÁTICA
   ├─ Cada 30 segundos:
   │  └─ ¿Hay items en cola? ✓
   │  └─ ¿Hay internet? ✓
   │  └─ Enviar a Firebase
   │
   └─ Si hay error:
      └─ Reintentar más tarde
      └─ Backoff exponencial (30s → 1m → 2m → ...)

4. DATOS LLEGAN A FIRESTORE
   └─ Se escriben en: businesses/1/branches/1/sales/{saleId}
   └─ Se aplican reglas de seguridad
   └─ Otros documentos: daily_reports, etc.

5. APP FLUTTER LO DETECTA
   └─ Snapshot listeners se activan
   └─ Los repositorios leen los datos
   └─ Las pantallas se actualizan automáticamente

6. USUARIO VE EL REPORTE
   └─ Desde cualquier lugar (web, móvil, escritorio)
   └─ Datos en tiempo real
   └─ Filtros por fecha, sucursal, cajero
   └─ Gráficos y tablas automáticas

│ 2. Ve reportes de todas sus sucursales                      │

┌─────────────────────────────────────────────────────────────┐

│ 2. Las ventas se registran en MariaDB local                │
│ 3. Cuando vuelve el internet, se sincronizan automáticamente │
